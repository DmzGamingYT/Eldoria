# Eldoria — Character Assets Credits

Eldoria uses the Quaternius CC0 Universal Humanoid Rig for the hero (Player.tsx)
and the 4 stationary NPCs (Npc.tsx). All three packs are CC0 upstream; one is
shipped, two are pending manual claim on itch.io.

### Local file
- `animations/UniversalAnimationLibrary_Male.glb` (~7.7 MB) — single GLB bundle,
  Standard variant. Carries **43 animation clips** on the male mannequin (195
  channels per clip, 67 nodes, 1 skinned mesh, 1 skin): A_TPose, Idle_FoldArms_Loop,
  Zombie_Walk_Fwd_Loop, Sword_Regular_A, Sword_Shield_Bash, Death_A, Wave, …
  This male bundle is the **canonical animation library** for the entire Eldoria
  cast: it works on the female skeleton too via the Universal-Humanoid bone
  names shared by all Quaternius packs.
- `animations/UniversalAnimationLibrary_Female.glb` (47.4 MB) — same bundle,
  female mannequin. The Standard variant ships the female skeleton as a
  *static rig* without its own clip set; in practice we drive it from the male
  bundle's clips. Kept on disk for completeness and as a fallback if a future
  Quaternius release ships gender-specific clips.

## Regeneration

To regenerate placeholder models: `node scripts/generate-placeholders.mjs`
These are simple multi-box humanoid shapes with color coding per archetype.

The bundles are versioned directly in git (no LFS yet). When the next CC0 pack
brings the total past ~150 MB, migrate `*.glb` to Git LFS via `.gitattributes`
to keep clones fast. Vite/Next both serve `public/` as static assets, so the
runtime cost is zero — only per-clone checkout time scales with binary size.
