docs(release): enrich GitHub Release body with per-platform download table

Ship des liens de téléchargement cliquables DIRECTEMENT dans la page
GitHub Release (au lieu de la liste d'assets brute par défaut). **Chaque
utilisateur voit immédiatement 8 liens explicites** quand il ouvre la
release v0.2.1+ — plus de recherche visuelle dans la liste d'assets.

**`.github/workflows/release.yml` step `softprops/action-gh-release@v2`**

### 🔧 Changements de workflow

- `generate_release_notes: true` → **`false`** — l'API GitHub fait passer
  l'auto-gen AVANT le `body:` (les mutuellement exclusif), donc pour que
  ma table soit visible je désactive l'auto.
- Template expression `${{ github.ref_name | replace: 'v', '' }}` appliquée
  partout où une version apparaît — yield `0.2.1` pour le tag `v0.2.1`,
  matchant l'`artifactName` d'electron-builder.
- **Mac ZIPs explicitement listés** : electron-builder pack déjà DMG + ZIP
  pour Mac (les deux cibles), je rends les ZIPs accessibles (utiles
  pour les power users / Homebrew Cask maintainers).

### 🔧 Aligné

- `package.json` version `0.2.0` → **`0.2.1`** : aligne le tag Git
  `v0.2.1` avec le `${version}` que electron-builder injecte dans
  l'`artifactName`. Sans ça, les filenames commençaient par `Eldoria-0.2.0-…`
  alors que le body templatait `0.2.1` → liens 404.
- `CHANGELOG.md` : nouvelle entrée `[0.2.1] — 2026-06-18 — Correctifs chaîne
  de release multiplateforme` — documente les 3 root causes (bash `cp -r`,
  .deb metadata, `node` absent CI) + l'ajout du body.

Après ce commit, on retaggera `v0.2.1` (delete + re-push) → release.yml
