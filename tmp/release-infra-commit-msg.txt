feat(release): matrix 3-OS release workflow + CHANGELOG + README downloads

Point de pivot — Eldoria passe du prototype navigateur à un **vrai client
de bureau installable** sur Windows, macOS et Linux. La CI release s'occupe
de produire les 3 installeurs à chaque tag v*.

### ✨ Nouveauté

**`.github/workflows/release.yml`** — nouveau workflow GitHub Actions
déclenché sur push de tag `v*` (ou `workflow_dispatch`).
Jobs :
- `build` (matrice 3 runneurs natifs windows-latest / macos-latest / ubuntu-latest)
  - bun install + prisma generate + next build (production standalone)
  - electron-builder --<target> --publish never
  - upload-artifact par OS
- `release` (Ubuntu, gated `if: startsWith(github.ref, 'refs/tags/')`)
  - download-artifact (tous les OS fusionnés)
  - softprops/action-gh-release@v2 attache les installeurs à la GitHub Release
  - generate_release_notes: true → changelog auto depuis les PRs mergées

### 📥 Documentation

**`CHANGELOG.md`** — Keep-A-Changelog 1.1.0, entrée v0.2.0 (2026-06-18).
Honeste sur la **non-signature** des installeurs (caveat SmartScreen + Gatekeeper).

**`README.md`** — nouvelle section `## 📥 Téléchargements` placée juste
avant `## 🚀 Démarrage rapide`. Contenu :
- tableau 3 OS (formats NSIS portable + DMG x64/arm64 + AppImage+deb+rpm)
- liens directs /releases/latest/download/ par plateforme
- snippet Linux AppImage (wget + chmod + exec)
- snippet Linux deb/rpm
- caveat signature v0.2.0 → v0.3.0
- snippet publish-a-new-release pour les mainteneurs

### 🔧 Détails techniques

- `concurrency.cancel-in-progress: false` → ne tue jamais un build en cours
- `prerelease: ${{ contains(github.ref_name, '-') }}` → RC tags auto-flaggés
- `permissions: contents: write` au top-level (job hérite)
- artifact glob `*.{exe,dmg,zip,AppImage,deb,rpm,blockmap}` couvre toutes les
  sorties electron-builder actuelles (pas de .pkg / .snap / .tar.gz)
- `fail_on_unmatched_files: true` → bloque la release si un format manque
- Signature macOS / Windows désactivée pour v0.2.0 → prévu v0.3.0 quand
  les secrets CSC_LINK / CSC_KEY_PASSWORD / Apple Developer ID seront dispos
